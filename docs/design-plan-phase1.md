# Summoner Atlas — Phase 1 设计规划

## 产品定位

**Summoner Atlas** 是一个 LOL 数据决策平台，帮助玩家在对局中快速找到英雄选择与玩法。品牌概念：**Hextech × Atlas（古代星图）**——数据不是冷冰冰的表格，而是可以被"探索"和"发现"的星图。

首批支持：国服海克斯大乱斗（核心）。

---

## 核心设计决策

### 场景优先级

| 场景     | 角色     | 时间窗口 | 设计含义                             |
| -------- | -------- | -------- | ------------------------------------ |
| 选人阶段 | **核心** | 15-60s   | 扫视优于阅读，信息密度高，交互零摩擦 |
| 加载界面 | 辅助     | 30-90s   | 可展示更深度的对局情报               |
| 死亡等待 | 辅助     | 碎片化   | 快速查单个装备/符文决策              |

### 交互模式

| 页面            | Push（推）                             | Search（搜）               |
| --------------- | -------------------------------------- | -------------------------- |
| 首页            | **70%** — 版本格局、强势英雄、趋势变化 | 30% — 全局搜索框           |
| 英雄详情/选人页 | 10% — 相关推荐                         | **90%** — 输入英雄名查数据 |

### 数据源策略（按模式绑定）

| 模式         | 数据源      |
| ------------ | ----------- |
| 海克斯大乱斗 | 仅国服      |
| 召唤师峡谷   | 国服 + 韩服 |
| 云顶之弈     | 国服为主    |
| 其他         | 全球默认    |

UI 上区域选择器跟随模式联动切换。

### 发光层级（信息层级语言）

越亮 = 越重要。三个层级：

- **L3 高强度**：Hero 核心数字、当前版本号、强势英雄排名数字
- **L2 中强度**：数据卡片边框、选中状态、主按钮 hover
- **L1 氛围**：区域分隔线、非活跃元素 hover

### 字体策略

| 用途               | 字体                                                              | 加载方式             |
| ------------------ | ----------------------------------------------------------------- | -------------------- |
| 英文标题（展示型） | Cinzel / 自定义衬线体（宽字距、大写、碑文感）                     | Google Fonts，子集化 |
| 英文正文           | Inter（数字等宽）                                                 | Google Fonts         |
| 中文/韩文          | 系统字体栈（PingFang SC / Microsoft YaHei / Apple SD Gothic Neo） | 零加载               |
| 代码/数据          | JetBrains Mono（后台页面）                                        | Google Fonts         |

### 动效策略

| 层级      | 范围                                                                | 工具                           |
| --------- | ------------------------------------------------------------------- | ------------------------------ |
| L1 微交互 | hover/active/focus、按钮反馈、骨架屏                                | CSS transition + Framer Motion |
| L2 叙事   | 首页 staggered reveal、Atlas 星图背景漂移、排名数字滚动、搜索框展开 | GSAP                           |
| L3 沉浸   | 英雄详情页大图过渡                                                  | GSAP ScrollTrigger             |

硬性约束：所有动效尊重 `prefers-reduced-motion`，GSAP 动画仅使用 transform/opacity。

## 首页模块规划（7 Sections）

### S1 — Hero

- **内容**：当前版本号 + 数据样本量 + 更新时间 + 全局搜索框 + 价值主张一行
- **布局**：全宽居中，最大字号，L3 高强度发光
- **背景**：Hextech 星图网格线缓慢漂移（GSAP 无限循环）
- **AIDA**：Attention

### S2 — 强势英雄 (stablePicks)

- **内容**：当前版本 Top 5-8 英雄（头像 + 胜率 + 场次 + 可信度徽章）
- **布局**：Bento Grid，不规则卡片大小（第一名最大）
- **图表**：水平条形图 + 辉光排名数字
- **AIDA**：Interest

### S3 — 趋势变化 (trendingUp / trendingDown)

- **内容**：胜率上升/下降英雄，带变化箭头 + 幅度数字
- **布局**：双列对比（上升 | 下降）
- **图表**：双向发散条形图
- **AIDA**：Interest

### S4 — 海克斯推荐 (researchAugments)

- **内容**：当前版本值得优先研究的 Top 5 海克斯 + 适配英雄头像
- **布局**：横向滚动卡片
- **图表**：径向条形图或迷你柱状图
- **AIDA**：Desire

### S5 — Atlas 探索区

- **内容**：英雄 × 海克斯协同关系静态预览（星图节点 + 连线 + 节点闪烁动画）
- **布局**：全宽交互预览区，点击进入独立 `/atlas` 页面
- **技术**：D3 force simulation（独立页），首页用静态 SVG + GSAP 微动效
- **AIDA**：Desire（品牌差异化核心）

### S6 — 模式入口 (modeData)

- **内容**：召唤师峡谷 / 云顶之弈 / 竞技场 快捷入口卡片
- **布局**：三列卡片，未开放模式标注"即将开放"
- **AIDA**：Action

### S7 — Footer

- **内容**：数据方法 / 来源声明 / 关于 / 支持 / Riot 免责声明
- **布局**：标准四列 Footer
- **AIDA**：信任锚点

## 阶段划分

### Phase 1（当前）— 骨架 + 首页 MVP

- shadcn/ui + Base UI 初始化 + Hextech 主题系统
- 首页完整 7 个 Section（Mock 数据驱动）
- 全局搜索框（首页嵌入式）
- 英雄详情页（单页滚动）
- 导航框架（桌面端顶部 + 移动端底部 TabBar）
- GSAP 动效：首页 staggered reveal + Atlas 星图背景
- 国际化路由 `/[lang]`
- Dark/Light 主题切换

### Phase 2 — 数据交互深化

- Ctrl+K 全屏搜索面板
- Recharts 图表全面接入
- 英雄对比页
- 海克斯榜 / 装备榜 / 符文榜独立页面
- 版本/Meta 页面
- 数据筛选器联动

### Phase 3 — Atlas 差异化 + 后端

- Atlas 探索区独立页面（D3 力导向网络图）
- 个人中心（OAuth 接入后）
- 后台管理
- ISR 缓存策略、图片优化

## 关键约束

1. 所有数据页面标注数据版本，比较只在同版本/同模式/同筛选条件下进行
2. 低样本高胜率必须明确降级标注，不可作为推荐主结论
3. 页面底部始终包含 Riot Games 免责声明
4. 与 Riot Games 无关联
5. 动效必须尊重 `prefers-reduced-motion`
6. 所有交互目标最小 48px（移动端）/ 无不可逆操作
